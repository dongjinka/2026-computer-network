## 목표

기존 TCP/IP가 극한 지연 환경에서 왜 잘 작동하지 않는지 설명한다.

이 역할은 프로젝트의 **문제 제기 파트**를 맡는다.

## 조사할 내용

- TCP/IP의 기본 동작 전제
    - end-to-end connection
    - ACK 기반 신뢰성
    - timeout과 retransmission
    - congestion control
- 극한 지연 환경의 특징
    - long delay
    - intermittent connectivity
    - high packet loss
    - network partition
- TCP/IP가 이런 환경에서 겪는 문제
    - ACK가 너무 늦게 도착함
    - 연결 유지가 어려움
    - 재전송이 비효율적임
    - 손실과 단순 지연을 구분하기 어려움
    - 전체 end-to-end 경로가 동시에 존재하지 않을 수 있음

## 정리해야 할 결과물

- TCP/IP 한계 비교표
- 극한 지연 환경 예시 2~3개
- 발표/보고서의 Introduction 또는 Background 초안

## 예시 산출물

| TCP/IP의 전제 | 극한 지연 환경에서의 문제 |
| --- | --- |
| 송신자와 수신자 사이의 연결이 유지됨 | 연결이 자주 끊기거나 전체 경로가 동시에 존재하지 않을 수 있음 |
| ACK가 일정 시간 안에 돌아옴 | 지연 시간이 너무 길어 ACK가 매우 늦게 도착함 |
| 손실은 재전송으로 복구 가능함 | 재전송 자체가 오래 걸려 비효율적임 |
| 패킷 손실은 혼잡으로 해석될 수 있음 | 실제로는 혼잡이 아니라 지연이나 단절일 수 있음 |

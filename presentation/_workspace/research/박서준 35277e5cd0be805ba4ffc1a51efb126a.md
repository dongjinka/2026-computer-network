## 담당 목표

DTN이 기존 TCP/IP의 한계를 어떤 방식으로 해결하는지 설명한다.

이 역할은 프로젝트의 핵심 이론 파트를 맡는다.

## 조사한 내용

- DTN의 정의
- Store-and-forward 방식
- Bundle Layer의 역할

- Custody transfer 또는 reliable forwarding 개념
- TCP/IP와 DTN의 구조적 차이

## 핵심 설명 방향

DTN은 송신자와 수신자가 동시에 연결되어 있지 않아도 데이터를 전달할 수 있도록 설계된 네트워크 구조이다.

중간 노드는 데이터를 바로 버리지 않고 저장해 두었다가, 다음 연결 기회가 생기면 다음 노드로 전달한다.

즉, DTN의 핵심은 **항상 연결되어 있어야 한다는 전제를 버리고, 연결 가능한 시점에 단계적으로 데이터를 전달하는 것**이다.

# DTN 데이터 전달 과정

1. Application이 데이터를 생성한다.
2. Bundle Layer가 데이터를 bundle 단위로 묶는다.
3. 현재 연결 가능한 노드에게 bundle을 전달한다.
4. 중간 노드는 bundle을 저장한다.
5. 다음 contact opportunity가 생기면 다음 노드로 전달한다.
6. 최종적으로 destination node에 도착한다.

### TCP/IP vs DTN 비교

| 항목 | TCP/IP | DTN |
| --- | --- | --- |
| 연결 방식 | End-to-end 연결 중심 | Hop-by-hop 저장 후 전달 |
| 경로 전제 | 전체 경로가 동시에 연결되어 있어야 함 | 전체 경로가 동시에 연결될 필요 없음 |
| 데이터 단위 | Segment / Packet | Bundle |
| 중간 노드 역할 | 주로 즉시 전달 | 저장 후 전달 |
| 적합 환경 | 안정적인 인터넷 환경 | 지연과 단절이 큰 환경 |
